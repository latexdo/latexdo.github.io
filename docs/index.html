About

What is LatexDo?

LatexDo is an open-source initiative for making serious LaTeX writing simpler, faster, and easier to maintain across web and desktop.

The project brings source editing, PDF preview, local builds, diagnostics, downloads, and documentation into one focused ecosystem for researchers, students, technical writers, and anyone preparing documents with LaTeX.

Open by design

LatexDo is meant to be inspectable, forkable, and shaped by the people who rely on LaTeX for real work.

Start here

Quick start

The fastest path is the hosted editor. Open the web editor in your browser when you want to draft or inspect LaTeX without preparing a local environment.

Use the browser editor

Go to editor.latexdo.org and start from a new document or an existing source file.

Terminal install

Install the CLI

The install script downloads the latexdo command to $HOME/.local/bin by default, then bootstraps the local LatexDo source and npm dependencies.

curl -fsSL https://latexdo.org/install.sh | bash

After installation, run latexdo. If your shell cannot find the command, add $HOME/.local/bin to your PATH.

Custom install directory

Set LATEXDO_BIN_DIR before running the installer to choose another destination.

Skip bootstrap

Set LATEXDO_SKIP_BOOTSTRAP=1 if you only want to install the CLI command first.

Requirements

Prepare local builds

Desktop and CLI workflows use your local tools. Install Node.js 20 or newer, npm, and a TeX distribution that includes latexmk.

  • macOS: install MacTeX or another TeX Live based distribution.
  • Windows: install MiKTeX or TeX Live.
  • Linux: install TeX Live from your distribution or upstream packages.

Installers

Use direct downloads

Desktop installers, update metadata, and checksums are published on the LatexDo downloads page. Use the manifest for update metadata and the SHA-256 list to verify files after download.

Workflow

Work on a LaTeX project

Keep your source files, bibliography, rendered PDF, build diagnostics, and review notes together. For existing projects, open the project folder locally so LatexDo can use the same files and TeX toolchain you already use from the terminal.

  1. Open or create the main .tex file.
  2. Attach related .bib files when the paper uses citations.
  3. Compile with your local TeX distribution and inspect diagnostics.
  4. Export or verify the final PDF before submission.

Support

Troubleshooting

The latexdo command is not found.

Add the install directory to your shell path. The default directory is $HOME/.local/bin.

Local compilation fails.

Confirm that your TeX distribution is installed and that latexmk is available in your terminal.

I need help with a project or install issue.

Email contact@latexdo.org or open an issue in the LatexDo repository.
